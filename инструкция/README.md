# Автогенерация страниц игр (Steam + SteamSpy)

## Что это
Скрипт раз в сутки тянет топ-100 игр по количеству игроков за 2 недели (SteamSpy),
дотягивает по каждой цену/описание/теги (Steam Store API) и генерирует
markdown-страницы в папку `games/`. GitHub Actions коммитит их автоматически —
свой сервер и Supabase не нужны.

## Установка в твой репозиторий

1. Скопируй в корень своего репо:
   - `scripts/generate-games.js`
   - `.github/workflows/update-games.yml`

2. Если сайт на Jekyll (стандартный движок GitHub Pages) — создай шаблон
   `_layouts/game.html`, чтобы markdown-файлы красиво оборачивались в дизайн сайта.
   Если сайт НЕ на Jekyll (чистый HTML/статик-генератор типа Astro/Hugo/11ty) —
   скажи какой, подгоню генератор под нужный формат (сразу .html вместо .md).

3. Первый запуск — вручную:
   - Открой вкладку **Actions** в репозитории на GitHub
   - Выбери workflow "Обновление страниц игр"
   - Нажми **Run workflow**

4. Дальше страницы будут обновляться сами каждый день в 03:00 UTC.

## Настройка

Все параметры — вверху `.github/workflows/update-games.yml`:
- `GAMES_LIMIT` — сколько игр брать (по умолчанию 100)
- `DELAY_MS` — пауза между запросами к Steam (не ставь меньше 1500, иначе бан по рейт-лимиту)
- частота обновления — строка `cron` в начале workflow-файла

## Важно про SEO
Не оставляй страницы как есть — просто вывод API Google посчитает
"тонким контентом". Стоит доработать шаблон `_layouts/game.html`, добавив:
- свой короткий редакторский комментарий/вердикт
- блок "похожие игры" (ссылки на другие сгенерированные страницы)
- хлебные крошки и навигацию по жанрам
